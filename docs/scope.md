# FluentOS Scope

## What FluentOS Covers

| Category | Scope |
|----------|-------|
| **Visual Design** | Desktop themes, icon packs, color schemes, typography guidelines |
| **Personalization** | Registry tweaks, system settings, Start Menu, Taskbar customization |
| **Tools & Integration** | Seelen (window manager), Windhawk (UX patcher), DWMBlurGlass (blur effects), Explorer tweaks |
| **Installation Automation** | PowerShell scripts for repeatable, minimal-friction setup |
| **Configuration Profiles** | JSON-based presets (light/dark, role-based for Business/Fleet) |
| **Documentation** | Vision, roadmap, requirements, limitations, legal framing |
| **Policy Guidelines** | Best practices for organization-wide deployments (Business/Fleet editions) |

## What FluentOS Does NOT Cover

| Category | Out of Scope |
|----------|-------|
| **Kernel-level changes** | Driver modifications, kernel patching, deep OS modifications |
| **Security bypasses** | Windows Defender circumvention, licensing evasion, unauthorized system access |
| **Windows binaries** | Custom ISO builds, redistribution of proprietary Microsoft components |
| **Non-Windows platforms** | macOS, Linux support (Windows 11 only for v0.1+) |
| **Hardware drivers** | GPU drivers, network adapters, chipset drivers (use Windows Update or vendor tools) |
| **Commercial licensing** | No license bypass – all usage assumes valid Windows 11 license |

## In Scope vs. Out of Scope Summary

### ✓ In Scope
- Harmless UX enhancements via community tools
- Registry and settings modifications via public APIs
- Automation via PowerShell and supported Windows features
- Documentation and best practices

### ✗ Out of Scope
- Anything requiring kernel access or driver modification
- Circumventing Microsoft licensing or security features
- Custom system binaries or OS components
- Unauthorized system patches or exploits
